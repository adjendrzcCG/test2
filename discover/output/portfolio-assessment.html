🗺️ Prioritized Modernization Roadmap

▼

🔴 Immediate Actions (0–3 months)

1
Replace WebSphere 7.0 on CRMApp-002 — EOL since 2015. Active security risk with no vendor patches for 10+ years.
2
Upgrade Windows Server 2012 on HRApp-004 — HR/payroll data on unpatched OS is a compliance violation.
3
Upgrade IIS 8.0 on HRApp-004 — Replace with IIS 10 on Windows 2022, or migrate to containerized Kestrel/Nginx.
4
Upgrade RHEL 7 on CRMApp-002 and AnalyticsApp-003 — Both OS instances require upgrade to RHEL 8/9.

🟡 Short-Term (3–12 months)

5
Full EOL stack upgrade for AnalyticsApp-003 — Python 3.9→3.12, Tomcat 6.1→10.x, PostgreSQL 13→16. Low risk, already containerized.
6
Containerize CRMApp-002 — Enable Kubernetes on AWS EKS for portability and easier future upgrades.
7
Clarify .NET Core version on HRApp-004 — If .NET Core 3.1 (EOL), upgrade to .NET 8 LTS immediately.
8
Upgrade Java 11 → 21 on CRMApp-002 — Move to current LTS for long-term security support.

🟢 Medium-Term (1–2 years)

9
Full cloud migration for HRApp-004 — Eliminate hybrid on-premise footprint and consolidate to AWS.
10
ERPApp-001: AIX → Linux OS migration — Move from proprietary IBM AIX to RHEL/Ubuntu. Eliminate AIX license and EOL risk.
11
Oracle 19c managed DB migration for ERPApp-001 — Plan migration to RDS Oracle or managed service before April 2027 EOL.

🔵 Long-Term (2–5 years)

12
ERPApp-001 COBOL modernization — Strangler-fig pattern to incrementally replace COBOL modules with modern services over 3-5 years.
13
SQL Server → PostgreSQL for HRApp-004 — High effort but eliminates SQL Server licensing cost long-term.
14
ARM/Graviton for CRMApp-002 and AnalyticsApp-003 — 20-40% compute cost reduction on AWS Graviton instances.
